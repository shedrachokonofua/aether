## 1. TL;DR

The cluster's problem is not namespace names. It is that **governance is a function of how a namespace was born, not of what it is.** Seven creation channels (committed tofu, applied-but-uncommitted tofu, three sibling-repo tofus, raw kubectl, helm `create_namespace`, operator side-effects, Kyverno clones) produce seven label postures, and every cross-cutting concern — PSA, netpol, backup, quota, priority, gateway attachment, alerts — is hand-plumbed per app. That is why backups silently rotted twice in five days (affine/immich dumping frozen pre-migration databases while dashboards showed green; mnemo failing ~hourly for 38+ runs), why the previous census missed two 60-day-old colony namespaces, why the password vault shares a secret-visibility domain with six mutable-tag apps, and why nearly every LLM provider key the owner has sits in one 14-app, ~20-pod (49-container) mega-namespace with two autonomous agents and an SSRF-by-design scraper.

The fix, in one sentence: **a namespace contract — 4 required labels (`tier`, `owner`, `backup`, `exposure`) plus a handful of tier-defaulted optionals — from which every runtime behavior is derived mechanically by Kyverno CEL policies + PSA + Cilium + a typed tofu module, enforced identically for namespaces this repo creates and namespaces it never sees.** Unknown namespaces land quarantined (default-deny both directions, zero cloned credentials, self-announcing) instead of open. Names change only where a data migration is already owed; there are a bounded set of moves worth paying for, and each one is executed as a backup-restore, so **the migration program is the restore-drill program**.

End state: ~66–68 small namespaces, one screen, four labels each (`kubectl get ns -L aether.shdr.ch/tier,aether.shdr.ch/backup,aether.shdr.ch/exposure,aether.shdr.ch/owner` tells the whole story). The vault, the LLM keys, the agents, the client site, and the internet's probe traffic each sit in their own blast radius. Pi saturation degrades in a chosen order. A sibling repo can create a namespace on a Saturday and it is governed, fenced, and visible by Sunday.

## 2. Design principles

1. **Namespace = blast radius.** It is the scope of secret visibility, PSA, NetworkPolicy, RBAC, quota, and atomic teardown (`delete ns` is the only uninstaller). It is not a folder, a team, or an environment.
2. **Names are immutable identity; labels are mutable attributes.** Names encode only stable function. No env, tier, tool, or arch in a name — ever. Pure renames are pure cost (PV rebinds, DNS literals, S3 key-prefix history forks, metric continuity); renames ride data migrations or don't happen.
3. **One decision per namespace.** Creating a namespace means declaring the contract labels. PSA, netpol, LimitRange/quota, priority, backup plumbing, gateway rights, pull-secret eligibility, arch affinity, and alert severity all derive. A label exists only if a named machine consumes it.
4. **Derivation is implemented once — at admission.** Kyverno derives; the tofu module only declares. Out-of-band creators (colony, osemu, shdrch, node-remediator, GitLab CI, operators) are first-class citizens governed by the same admission path. Convention already failed: the census missed colony-dev for 57 days.
5. **One writer per surface.** Tofu writes Namespaces + contract labels + app-specific allows. Kyverno writes derived objects (`synchronize: true`). Nothing is written by two systems.
6. **Fail-closed on security invariants, fail-open on availability paths.** Unknown namespaces get default-deny and no credentials. Pod-path mutations (priority, arch, kata default) use `failurePolicy: Ignore` so a Kyverno outage can't stop scheduling. PSA and two VAPs run in-process in the apiserver as the engine-independent floor that survives a Kyverno outage — and that holds during waves 0–1 of a rebuild-from-zero, before Kyverno exists.
7. **Backups are a namespace property, checked at admission, proven by restore.** Schedule-green ≠ restore-green: the cluster had green jobs backing up dead databases this week. A `backup=critical` namespace *is* backed up with its own scoped credentials, or admission says why not.
8. **Secret scope is the sharpest splitter.** Any pod in a namespace can mount any Secret in it. If two workloads must not read each other's credentials, they do not share a namespace. This rule dissolves `infra` and `personal` — fully, not "opportunistically."
9. **The relief valve is explicit.** Four PriorityClasses + LimitRanges everywhere make "saturate the Pis" a policy instead of a gamble. Agents and sandboxes evict before Immich or the gateway, never the reverse.
10. **Idle control planes are debt.** 23 knative pods serving zero ksvc are CVE surface and RAM the Pis subsidize. Delete until a consumer exists; reinstalling is one apply.

## 3. Taxonomy

### 3.1 Six tiers (`aether.shdr.ch/tier`)

| Tier | Definition | Target members |
|---|---|---|
| **platform** | Cluster machinery: CNI, CSI, PKI, policy engines, operators, observability, GPU enablement, CI runners, remediation. PSA per-ns explicit — privileged only where CSI/CNI/eBPF/hostNet demands it, each carrying a `privileged-justification` annotation. | kube-system, kube-public, kube-node-lease, default (gateway residue, otherwise empty), system, ceph-csi-cephfs, observability (new), gpu-system (new), cert-manager, istio-system, cilium-secrets, kyverno, cnpg-system, crossplane-system, tetragon, trivy-system, policy-reporter, goldilocks, headlamp, gitlab-agent, gitlab-runner, agent-sandbox-system, node-healthcheck-operator-system, aether-k8s-node-remediator, medik8s-leases, wasmcloud-system, ups (new) |
| **app** | A workload the owner or family uses. Baseline PSA ratcheting to restricted; default-deny + explicit allows; backup label mandatory. One namespace per app stack. | affine, immich, nextcloud, matrix, media, jellyfin (new), qbittorrent (new), dawarich, karakeep, miniflux, hoppscotch, temporal, yourspotify, open-design, games, and the ex-pool apps: litellm, openwebui, searxng, firecrawl, ai-serving, mnemo, vaultwarden, composer, orion, vane, memos, bentopdf, snapotter |
| **agent** | Autonomous or arbitrary-code workloads holding **standing credentials**: AI agents, dev workspaces, session controllers. Restricted PSA (kata via exemption where pod specs demand more), **FQDN-allowlisted egress**, no cloned registry credentials, batch priority. | hermes, holyclaude, mux, jupyter, coder, deskplane |
| **sandbox** | Ephemeral untrusted execution. **Never holds standing credentials** — that is the line between agent and sandbox, and it is why they are separate tiers. Kata **required** (mutate + validate-reject), deny-all + DNS + declared tool endpoints, quota + LimitRange, lowest priority, TTL-eligible. | sandboxes, colony-sandboxes-dev |
| **guest** | Workloads owned by another repo or party, running on the shared kernel, deployed by their own IaC/CI. Governed by contract, not trust: quota, egress fence, no cloned creds by default, own alert route (daily digest + backup pages). | colony-dev, osemu-ehis-farms, shdrch, **globalping** |
| **tenant** | A real second party needing its own control plane. Always a vcluster. Host provides quota, egress+ingress fence, scoped backup credentials, host-side snapshots; tenant owns its interior. | vc-seven30 |

A seventh value, **`unclassified`**, is never declared — it is stamped by the quarantine machinery onto any namespace that arrives without a contract (§6.3).

Deliberate tier-set choices: no `ci` tier (gitlab-runner is one namespace; it gets platform tier + a per-ns quota override — a tier of one member is taxonomy for its own sake); no `operator`/`structural` sub-tiers of platform (PSA is per-ns explicit within platform anyway); **coder and deskplane are agent tier now**, not app-with-a-watch-note — coder runs arbitrary-code workspaces (148Gi) with standing creds, and deskplane's SA holds namespace-secrets CRUD plus cluster-wide nodes read; **globalping is guest**, not app — it executes *the internet's* measurement requests with `hostNetwork: true` on the flat LAN, L2-adjacent to Ceph.

### 3.2 What forces a namespace boundary (split when ANY differ)

1. **PSA level** — one privileged pod drags the whole namespace to privileged. Live example ×2: jellyfin's privileged FUSE rclone sidecar AND qbittorrent's gluetun `NET_ADMIN` sidecar (qbittorrent.tf:135-137) jointly keep all 12 media apps privileged. **Both** must leave for media to reach baseline — splitting jellyfin alone does nothing for PSS, because baseline forbids added NET_ADMIN.
2. **Secret visibility** — litellm's provider keys vs. hermes/mux/jupyter/firecrawl; vaultwarden vs. everything.
3. **Egress posture** — FQDN-allowlisted agents can't share a default with open-internet apps.
4. **RBAC principal scope** — an SA needing cluster or ns-wide grants defines its own trust domain (hermes-tungsten, coder, deskplane).
5. **Lifecycle / teardown atomicity** — things that die together live together.
6. **Quota need** — bursty/untrusted gets fenced without taxing neighbors.
7. **Owner repo** — different deployer = different namespace, always.
8. **Conflicting CRDs / operators / untrusted cluster-admin** — not a namespace split; that is the vcluster line.

Do **not** split: tightly-coupled stacks that share secrets and deploy/roll back as one unit (nextcloud+onlyoffice, matrix+bridges, an app + its CNPG + its redis + bootstrap jobs, the *arr stack minus the two privilege-holders). Do not split for versions/variants (labels) or aesthetics.

### 3.3 Deterministic placement rule for any future workload

Apply top-down; first match wins. This goes verbatim into `docs/` and the module README.

```
1. Needs its own API server, conflicting CRDs/operators, or untrusted
   cluster-admin?                                → vcluster: ns vc-<tenant>, tier=tenant
2. Deployed by a repo/party other than aether?   → tier=guest, their name, contract
                                                   labels enforced at admission
3. Ephemeral untrusted code execution, never
   holding standing credentials?                 → tier=sandbox (Sandbox CR in
                                                   `sandboxes`, or ephemeral ns + TTL)
4. Autonomous agent / arbitrary-code workspace
   holding credentials (LLM keys, tokens,
   browser sessions, kubeconfig-adjacent RBAC)?  → tier=agent, its own ns
5. Cluster machinery (storage/network/certs/
   policy/observability/CI/GPU)?                 → tier=platform, upstream default name
6. Otherwise: an app                             → ITS OWN namespace via
                                                   module "namespace", named for its
                                                   stable function. No pool additions,
                                                   ever — infra and personal are being
                                                   deleted, not grandfathered.
```

## 4. Naming convention & rename verdicts

### 4.1 Rules (validated at admission for `owner=aether` namespaces)

- **Shape:** flat, lowercase, function-named: `^[a-z]([a-z0-9-]{0,30}[a-z0-9])?$` (RFC 1123 subset, ≤32 chars, no trailing hyphen — keeps CNPG/vcluster synthesized names under limits). **Validated on CREATE only**, with a frozen grandfather list for pre-existing aether-owned names (at minimum `node-healthcheck-operator-system`, which sits exactly at the 32-char cap) — otherwise the Phase 1 relabeling of an existing namespace would trip the very rule this doc ships.
- **Reserved:** `kube-*` (Kubernetes), `vc-*` (vcluster host namespaces), `aether-k8s-*` (in-house cluster controllers). `-system` suffix is reserved for third-party operators keeping their upstream default name (cnpg-system, crossplane-system, …) plus the two new first-party exceptions below — never minted for apps.
- **No env suffixes** (`-dev|-stage|-staging|-prod|-test`) on aether-owned namespaces — this is a single-env cluster; environment belongs at the cluster/vcluster boundary. Guest repos own their names within the label contract (colony-dev's suffix is colony's choice and colony's cost; grandfathered).
- **Function over implementation** for platform subsystems (`observability` not `otel`, `gpu-system` not `nvidia`) so tool swaps are label-cheap. Apps keep app names — an app swap is a new app.

### 4.2 Rename verdicts — the honest ledger

Zero pure renames. Not conservatism — arithmetic. The gap between the live names and the greenfield ideal is labels, machinery, and a bounded set of splits, not names. Costs verified in this repo:

| Verdict | Namespaces | Why |
|---|---|---|
| **Never rename** | `system`, `ceph-csi-cephfs` | Hardest lock in the cluster: the `ceph-rbd`/`cephfs` StorageClass **parameters embed the CSI secret namespaces and SC parameters are immutable**; every one of the ~90 bound ceph PVs carries provisioner/nodeStage secretRefs pointing into them. A rename = new SC + secret-ref surgery on every volume. The grab-bag problem in `system` is solved by **splitting out** observability/GPU (stateless), never by renaming the storage core. |
| **Never rename** | `vc-seven30` | Syncer-encoded service names (`caddy-ingress-x-default-x-seven30`), OTel rewrite regex `^vc-seven30($|-)`, tenant API VIP 10.0.3.21. |
| **Never rename** | `kube-*`, `istio-system`, upstream operator names | Webhooks, chart defaults, community runbooks, hardcoded otel scrape target (`otel_collector.tf:240`). |
| **Never move** | `default/main-gateway` | Load-bearing across three codebases (61 in-repo parentRefs + colony's 4 routes + the vcluster syncer route) + VIP 10.0.3.19 in two Caddyfiles + AdGuard. Governance comes from `allowedRoutes: Selector` + hostname pinning + validation (§5), which requires no move. `default` is emptied of workloads and stays that way. |
| **Dissolve, not rename** | `infra`, `personal` | The names aren't the problem; the secret-visibility domains are. App-by-app extraction; the old names die empty and get deleted. |
| **Split, keep name** | `media`, `system` | media loses jellyfin + qbittorrent (both, §3.2) and drops to baseline; system loses observability + GPU and narrows to storage-CSI-only. |
| **Everything else** | keep | No live name encodes a volatile fact badly enough to pay for surgery. |

Also rejected: the 06-21 doc's forward prefixes (`pool-*`, `app-*`, `sbx-*`) even for new namespaces — prefixes encode tier in the name, violating principle 2; the tier label already carries it and a prefix is one more thing out-of-band creators get wrong.

## 5. The namespace contract

**Prefix:** one owned prefix, **`aether.shdr.ch/`** (DNS-owned, matching colony's `colony.shdr.ch/*` pattern; `db_backups.tf` already uses `aether.shdr.ch/component`). The legacy `aether.sh/arm-ok` pod key is grandfathered until the Kyverno CEL rewrite ships, then migrated. Upstream keys (`app.kubernetes.io/*`, `pod-security.kubernetes.io/*`, `istio.io/*`, `goldilocks.fairwinds.com/*`) stay as their owners define them.

### 5.1 Owner-set labels

**Required (the contract — typed variables in the tofu module, validated at admission):**

| Label | Values | Consumed by |
|---|---|---|
| `aether.shdr.ch/tier` | platform \| app \| agent \| sandbox \| guest \| tenant | every derivation in §5.3 |
| `aether.shdr.ch/owner` | aether \| colony \| osemu-ehis-farms \| shdrch \| node-remediator \| seven30 \| … | provenance (the colony-census-miss killer); quarterly attrition sweep; alert annotations |
| `aether.shdr.ch/backup` | critical \| standard \| none | module-provisioned ObjectStore/ScheduledBackup + scoped S3 identity; CNPG admission rule; snapshot schedules; prevent_destroy; delete-guard; freshness alerting |
| `aether.shdr.ch/exposure` | public \| internal \| tunnel \| lan-vip \| none | gateway `allowedRoutes` selectors; HTTPRoute hostname validation; cloudflared gating; LB-service allowance |

**Optional (tier-defaulted by the stamper; declare only to override):**

| Label | Values (default) | Consumed by |
|---|---|---|
| `aether.shdr.ch/egress` | cluster \| **internal** \| internet \| allowlist \| none (tier default) | netpol template selection; toFQDNs CNP for `allowlist`. **`internal` = cluster + home-LAN services; `internet` = world minus RFC1918.** The distinction matters here specifically: hermes has live hairpin dependencies on `*.home.shdr.ch` (LAN Caddy — hermes.tf:15/18/20-22), so `internet` would break it at flip — it needs `internal`-plus-allowlist until each hairpin is converted (§8.2). (holyclaude's only `*.home.shdr.ch` reference in code is its own ingress host; verify its in-container config before assuming LAN egress needs.) Tier defaults: app=internal, agent=allowlist, sandbox=none, guest=internal (globalping overrides to internet), tenant=existing CNPs, platform=staged-later. |
| `aether.shdr.ch/arch` | any \| amd64 \| arm64-preferred (default any) | Kyverno mutate → nodeAffinity (replaces the arch-labeler webhook). `arm64-preferred` = *preferred* Pi affinity so amd64 stays the relief valve. |
| `aether.shdr.ch/criticality` | high \| normal \| low (tier default: platform=high, app/agent/tenant=normal, sandbox/guest=low) | alert severity routing via `kube_namespace_labels` join |
| `aether.shdr.ch/lifecycle` | persistent \| ephemeral (default persistent) | ephemeral ⇒ TTL required + CleanupPolicy; persistent+critical ⇒ delete-guard (§5.3 row 14) |
| `aether.shdr.ch/registry-access` | dockerhub \| none (tier default: platform/app=dockerhub; agent/sandbox/guest/tenant/unclassified=**none**) | gates the pull-secret clone. **Explicit `dockerhub` overrides on agent namespaces that verifiably pull Docker Hub today: hermes (`nousresearch/hermes-agent:latest` + `busybox:latest`), holyclaude (`coderluii/holyclaude:latest`), mux (`node:22-bookworm`, `caddy:2-alpine`), deskplane (`kasmweb/*:1.17.0` session images)** — without these, their next pod reschedule after the Phase 2 gate hits anonymous docker.io rate limits at 3am. (Mirroring those images to the GitLab registry retires each override.) Structural namespaces on the exempt list (kube-*, cilium-secrets, medik8s-leases) are stamped `registry-access=none` explicitly — that is the mechanism by which kube-public/kube-node-lease lose the secret despite the platform-tier default |
| `aether.shdr.ch/runtime` | runc \| kata (default: runc; sandbox tier forced kata) | RuntimeClass mutate + validate |
| `aether.shdr.ch/mesh` | ambient \| off (default off) | passthrough to `istio.io/dataplane-mode`. **Not derived from tier**: ambient covers 2/54 namespaces today — deriving mesh would imply an enforcement layer that doesn't exist. Revisit when rollout resumes. |

**Derived (never hand-set; stamped by Kyverno):** `pod-security.kubernetes.io/{enforce,warn,audit}`, `goldilocks.fairwinds.com/enabled`, `istio.io/dataplane-mode`, `aether.shdr.ch/gateway-access[-public]`.

**Escape hatch (rare, audited):** `aether.shdr.ch/pss-override: privileged|baseline` — honored only when a VAP confirms `tier=platform` or a frozen grandfather list; every use is a standing PolicyReport entry. Steady-state target ≤ 9 (CSI ×2, istio-cni, tetragon, kepler→observability, trivy, gitlab-runner, games, globalping), plus a **dated transitional list, distinct from the steady-state target**: media until the Phase 5 split, and **holyclaude** — it carries a live tofu-owned `enforce=privileged` label (holyclaude.tf:18) with target tier=agent, so it must be grandfathered until the kata-exemption relabel lands or Phase 1's fail-closed VAP would reject the very label-adoption update Phase 1 performs on it.

**Annotations (prose, never selectors):** `aether.shdr.ch/description`; `aether.shdr.ch/source` (tf file or sibling repo URL); `aether.shdr.ch/hostnames` (comma-separated allowlist — every HTTPRoute host in the ns must be ⊆ it); `aether.shdr.ch/privileged-justification` (mandatory with pss-override); `aether.shdr.ch/ttl` (mandatory iff ephemeral); `aether.shdr.ch/data=rebuildable` (declares why backup=none is fine for a PVC-holding ns).

Rejected from the schema: a separate `data-class` label (zero-trust carried both tier and data-class feeding only alert routing — two labels doing overlapping work is a tired-operator mis-pick trap; `criticality` covers it) and a hand-set `restore-wave` label (gitops-sre; its own tier table showed the mapping is ~bijective — restore order **derives from tier** + the tofu DAG, §5.3 row 15 / §8.13).

### 5.2 The tofu module (creation-side typing)

```hcl
module "namespace" {
  source      = "../modules/namespace"
  name        = "immich"
  tier        = "app"            # enum-validated at plan time
  backup      = "critical"
  exposure    = "internal"
  hostnames   = ["immich.home.shdr.ch"]
  description = "Photo library"
  source_file = "tofu/home/kubernetes/immich.tf"
  # optional: egress, arch, criticality, lifecycle, registry_access, runtime, mesh
}
```

The module emits: (a) the Namespace with contract labels + annotations; (b) when `backup != "none"`, the **per-namespace SeaweedFS S3 identity** (scoped to `s3://aether-db-dumps/<ns>/*`), its credentials Secret, and the CNPG-I Barman-Cloud `ObjectStore` + `ScheduledBackup` CRs; (c) `prevent_destroy` on the namespace's stateful resources when `backup=critical`; (d) nothing else — netpol/LimitRange/quota baselines belong to Kyverno so out-of-band namespaces get identical treatment. **Adoption mechanics, two classes:** the 33 existing `kubernetes_namespace_v1` resources adopt the module via `moved` blocks (state-only); the 7 `helm_release.create_namespace = true` namespaces (kyverno, cert-manager, headlamp, crossplane-system, knative-operator, gitlab-agent, wasmcloud-system) were **never in tofu state, so `moved` cannot adopt them — they need `import` blocks**, otherwise plan shows CREATE and apply fails on AlreadyExists; their releases then flip to `create_namespace=false` and consume a module instance. **Adoption label-preservation rule:** each module instance must declare the namespace's current live label VALUES — e.g. infra carries `istio.io/dataplane-mode: ambient` (namespaces.tf:22, live) and a mesh-default-off module would silently strip it, dropping litellm/openwebui/hermes out of L4 mTLS and darkening the istio telemetry pipeline while still passing a "labels-only plan" gate (a label *removal* is a labels-only change). Same threat class: `goldilocks.fairwinds.com/enabled`, holyclaude's tofu-owned PSA label. A pre-adoption script diffs module-rendered labels against live labels and **blocks on any removal**.

### 5.4 The database sub-contract (CNPG)

Standardize now, while the CNPG estate is days old and carries zero backup debt: **Barman Cloud CNPG-I plugin from day one** (in-tree `barmanObjectStore` is deprecated in 1.26, removed in 1.30 — the cluster runs 1.29.1; adopting the plugin now means never carrying that migration). One `ObjectStore` per namespace, module-provisioned, per-ns credentials, `serverName` = cluster name — history keys to the app, not hand-typed prefixes, and survives any future move via `bootstrap.recovery`. SeaweedFS hardening in the same stroke: bucket **versioning + object-lock + lifecycle** expiry a few days past Barman retention (today: single admin identity, no lifecycle, unbounded growth, and the owner's own backup history includes a "self-delete defused" incident). Keep `cnpg-require-ceph-rbd-storage` (ported to CEL). CSI snapshots **exclude CNPG PVCs** — snapshot-without-fencing is unsafe; Postgres restores only via `bootstrap.recovery`. And the **dual-DB-window rule**: the CNPG-backup admission rule at Enforce PLUS "legacy postgres STS deleted within 7 days of verified cutover" — the affine/immich silent-data-loss class was a dual-DB-window bug, and four cutovers (litellm, openwebui, matrix, nextcloud) are still pending; without this rule it recurs at each one.

**Every hardcoded namespace exclusion array dies in the CEL port.** The 3 Kyverno policies and the arch-labeler webhook selector each carry hand-maintained lists that already silently omit the 9 post-census namespaces (7 new since 06-21 + the 2 census-missed colony namespaces); they become `namespaceSelector: tier in (platform)` — self-maintaining forever.

### 6.2 Rollout discipline

Every validate policy ships Audit → (fix findings, ≥1 week clean PolicyReports) → Enforce, per policy. Netpol per namespace: generate with observation first, flip default-deny per-ns with Hubble evidence. PSA: warn/audit=restricted everywhere immediately (pure signal); enforce ratchets per-ns — **only after the PSS-blocked-pods alert (§8.1) is live**, because enforce evaluates Pods, not Deployments, and this cluster has two live instances of silent 0/1 rejection.

### 6.3 The quarantine (`tier=unclassified`) — census blindness becomes an alert

Any namespace that arrives without contract labels (a new sibling repo, a helm chart, an operator) is mutate-stamped `tier=unclassified` and generated: PSA baseline + **default-deny both directions** + **zero cloned credentials** + `aether-sandbox` priority + a standing PolicyReport violation + a weekly live-vs-declared diff (module list + known-external registry) posted to ntfy. Quarantined namespaces **run** (nothing breaks at 3am) but cannot reach the network, the gateway, or any shared credential until classified. colony-dev sat invisible for 57 days; the next stranger self-announces in minutes and is fenced while it waits. Deliberately NOT the guest tier: an unattributed namespace must never inherit guest defaults (guest can be granted registry access; unclassified gets nothing, ever).

**How quarantine reconciles with the CREATE gates (§6.1 VAP b, §6.4.3 Enforce):** mutating admission runs before validation, so with Kyverno healthy an unlabeled namespace is stamped `tier=unclassified` *first* and **`unclassified` is a passing value for both the VAP and require-labels** — the namespace is admitted, quarantined, and self-announcing. With Kyverno down, the stamper never fires and the VAP fail-closes: CREATE is denied. Degraded mode is deny; normal mode is quarantine — neither gate is dead code and neither is vacuous. A creator self-declaring `tier=unclassified` is harmless: it is self-fencing. **Sequencing guard: quarantine's *generate* rules (the default-deny fence) have no Audit mode — they activate only after all 54 live namespaces carry contract labels (end of Phase 1)**, so the fence can never land on a not-yet-labeled infra or personal mid-migration; the mutate stamp and the PolicyReport/ntfy announcement are live from day one.

### 6.4 Governing the seven deployment channels

1. **This repo, committed tofu** — module-typed. CI fails if `kubernetes_namespace_v1` appears outside the module or any `helm_release` sets `create_namespace=true`.
2. **This repo, out-of-band-of-committed-tofu — two sub-channels, and the second is the dangerous one.** (a) *Applied-but-uncommitted tf files* (holyclaude.tf, orion.tf, db_backups.tf until 07-02, when they were committed mid-program; a rolling set of modified files remains) — banned as a standing state: Phase 0 commits the remainder; nightly CI `tofu plan -detailed-exitcode` + working-tree drift alert catches divergence within 24h. (b) ***kubectl-applied resources with no tf at all*** — the largest live drift is invisible to any working-tree check: **6 of the 12 live CNPG clusters (affine, immich, litellm, openwebui, matrix, nextcloud) exist in no tf file**, and the live affine/immich deployments already point at `*-cnpg-rw` services while affine.tf/immich.tf still render the legacy STS hosts. A full-root `tofu apply` would silently revert both apps to frozen pre-migration databases. Phase 0 step zero (§9) reconciles this class before any apply; the nightly `tofu plan` job is the standing detector for both sub-channels. The cluster's source of truth must not be one laptop's working tree — or one session's kubectl.
3. **Sibling-repo tofu** (colony, osemu-ehis-farms, shdrch) — governed at admission: stamper fills what they omit, quarantine fences what it can't classify. Follow-up: a ~10-line PR to each sibling adding the 4 required labels (colony already labels well; it needs only `tier` + `owner`; its `colony.shdr.ch/*` keys stay its own business). The contract interface is `docs/namespace-contract.md`. 30-day grace (VAP b and require-labels in Audit throughout), then Enforce on CREATE — flipped only after the sibling PRs merge (§6.1).
4. **Sibling repo via raw kubectl** (aether-k8s-node-remediator) — same admission coverage; resolve the split-brain by **adopting** the ns + deployment into this repo (it already owns the RBAC half via `node_remediation.tf:27`).
5. **Operator side-effects** (medik8s-leases, cilium-secrets) — exempt list, tier=platform, stamped by mutate.
6. **vcluster interior** — invisible by design; the host contract applies to `vc-seven30` itself (quota, egress+ingress CNPs, snapshots, offered backup creds).
7. **Kyverno-generated** — the generator is the contract.

**The residual hole is CI's god-token, and it makes every admission control advisory until fixed.** `gitlab-agent/gitlab-agent` and `headlamp/headlamp` hold cluster-admin CRBs; `gitlab-agent-ocs-scanning-pod-sa` separately reads all secrets cluster-wide (~450 today; the count churns with runner ephemera). Scope the agent to per-project impersonation with an allowlist ClusterRole that **explicitly excludes write on `kyverno.io` and `admissionregistration.k8s.io`** — otherwise CI can delete the very policies that constitute the contract. Headlamp flips to OIDC user impersonation (the `admin`-group CRB already exists for humans). Sequenced in Phase 2; test on colony's project first.

## 8. Practices you did not know to ask for

*(Each specific to this cluster's live state, 2026-07-02.)*

1. **PSA rejections are silent, and it has already bitten you twice.** `enforce` evaluates Pods, not Deployments — the Deployment admits fine and pods just never appear. `default/globalping-probe` sat 0/1 for 5 days; `personal/holyclaude` for ~2.5 days. Ship one alert in Phase 1, **before any enforce flip**: `kube_replicaset_status_replicas − kube_replicaset_status_ready_replicas > 0 for 15m` joined with FailedCreate events → page. Without it every PSA ratchet in this plan is a foot-gun; with it, ratcheting is safe.
2. **The hairpin defeats every control you're about to build.** affine, onlyoffice, orion, vane, and hermes call litellm/searxng via `https://*.home.shdr.ch` — traffic exits the cluster to the Caddy VM (your documented single-router 502 SPOF) and comes back as LAN ingress, invisible to netpol and mesh identity. Standardize in-cluster calls on cluster DNS **as part of each app's move checklist** — it's also why agent FQDN allowlists must account for LAN hostnames until each hairpin is converted.
3. **One Talos machine-config line makes your kata namespaces restricted instead of privileged.** PSA exemption `runtimeClassNames: [kata]`: the VM boundary substitutes for the PSS controls, so holyclaude/mux/sandboxes carry `enforce=restricted` while kata pods keep their capabilities inside the guest. Your riskiest namespace becomes your most locked-down without touching the workload.
4. **Your backup layer has a god-key, and ransomware knows this playbook.** The SeaweedFS **admin** keypair is cloned into 11 namespaces as `db-backup-s3`; any secret-reading compromise in any of them can delete every namespace's dumps. Per-ns scoped identities (module-provisioned) + bucket versioning + object-lock + lifecycle kill the god-key, the stale-target bug class, and unbounded dump growth in one mechanism.
5. **Hostname squatting is your top unpriced risk, and conflict resolution is "first-created wins."** All 8 listeners are `allowedRoutes: All` with 0 ReferenceGrants — any namespace can today claim `vaultwarden.home.shdr.ch` or attach to the public `nextcloud.shdr.ch` listener; Gateway API resolves conflicts by creation timestamp, so a route created during any gap wins silently. Three cheap layers: Selector attachment, pinned listener hostnames, hostname-⊆-allowlist validation. And the second ingress path is governed the same way: cloudflared images run only in `exposure=tunnel` namespaces.
6. **Make the mnemo bug an admission error and the affine/immich bug a policy.** "CNPG Cluster in a backup≠none namespace must declare backup" (Enforce) turns 38 silent hourly failures into a rejected apply. "Legacy STS deleted within 7 days of verified cutover" closes the dual-DB window that produced two green-but-worthless backup streams — with four cutovers still pending, this is not hypothetical.
7. **The CNPG cutover window is the cheapest namespace-move window you will ever get.** litellm/openwebui CNPGs are pre-staged and **idle right now**; a `bootstrap.recovery` restore into a *new* namespace costs the same as a cutover in place. Ride the window: litellm and openwebui cut over directly INTO their target namespaces, making the infra dissolution and the DB migration one operation. Miss it and you pay a second migration you will never schedule.
8. **Ceph-RBD file data has no file-level backup at all — zero VolumeSnapshot CRDs exist.** Install external-snapshotter + a VolumeSnapshotClass; drive nightly snapshots from the backup label for the small high-sensitivity RWO volumes (vaultwarden 5Gi, synapse media + bridge sessions 30Gi, karakeep, dawarich, hermes/mux workspaces) **and vc-seven30's 163Gi** (crash-consistent tenant protection without host visibility). **Exclude CNPG instance PVCs** — snapshot-without-fencing is unsafe; Postgres restores only via `bootstrap.recovery`.
9. **prevent_destroy guards the wrong things.** The protection set is stale: several of the 18 blocks guard legacy postgres PVCs about to become garbage (affine/firecrawl/hoppscotch/miniflux legacy pg, matrix pg pre-cutover), while immich (2Ti), mux (200Gi), and 11 of 12 CNPG clusters have nothing. Re-key it to `backup=critical` via the module; strip it from legacy PVCs as they retire.
10. **Orphaned RBAC is privilege resurrection, not clutter.** Nine CRBs still bind SAs in deleted namespaces (`medik8s-system` ×6 — node-healthcheck ×3 + self-node-remediation ×3, `olm`, `kubero-operator-system` ×2) plus a leftover medik8s mutating webhook still intercepting API calls, plus step-issuer bound to `cert-manager/default`. Anyone who recreates those ns+SA names silently inherits the grants. Sweep in Phase 0, then keep a **standing Audit-mode policy flagging ClusterRoleBindings whose subjects reference nonexistent namespaces** — the class, not the instances.
11. **Digest-pin exactly where it pays.** Half the cluster runs `:latest`; a cluster-wide tag gate is instant self-DoS. Scope an ImageValidatingPolicy (digest-or-immutable-tag) to `backup=critical` namespaces only — temporal serves from `auto-setup:latest`, matrix runs `:latest` in all four containers, osemu runs `wordpress:latest` at revision 20. Everything else gets a warn-only `:latest` report.
12. **One kube-state-metrics flag makes the whole scheme queryable — otherwise it's invisible.** `--metric-labels-allowlist=namespaces=[aether.shdr.ch/tier,…owner,…backup,…criticality]` turns contract labels into PromQL join keys: tier-routed alert severity (~4 routes), backup-freshness-by-contract, Kepler watts-per-tier showback on fixed hardware. Add the same keys to OTel `k8sattributes` once; traces and logs carry them forever.
13. **Restore order is derived, not remembered — and your migration IS the drill program.** `task dr:plan` renders live label state into an ordered rebuild checklist (an etcd snapshot restores the control plane, *not* your applications). Because every move in Phase 4 executes as `bootstrap.recovery` or snapshot-restore into the new namespace, by the end of the program you will have proven restore on every critical datastore without scheduling one artificial drill. Quarterly thereafter: one random critical-ns restore + one CNPG PITR.
14. **The delete-guard is two hours of work and one averted catastrophe.** Namespace DELETE on persistent+critical namespaces requires an explicit same-change annotation. `kubectl delete ns vaultwarden` at 1am should be a conversation, not an outcome.
15. **VAP is the answer to "who guards the guard."** Kyverno's resource webhook is fail-closed — good — but that makes Kyverno an availability dependency and a rebuild-ordering problem. The two VAP invariants run in-process in the apiserver: they hold during a Kyverno outage and during rebuild waves before Kyverno exists. Twenty lines of CEL removes the single point of policy failure.
16. **ESO + your existing OpenBao is the secrets end-state — after boundaries.** You already run Bao with per-project transit engines (the osemu pattern). external-secrets-operator with **per-namespace SecretStores** bound to per-namespace Bao policies makes secret provenance namespace-scoped and auditable; disable ClusterSecretStore outright. Sequenced last (Phase 6) deliberately: boundaries first, then identity-scoped secrets.
